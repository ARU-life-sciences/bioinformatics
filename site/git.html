git and GitHub

Two very important technologies for modern software development. For students and academics, these are invaluable tools to catalogue and develop your code. Many branches of academia which rely on software, like bioinformatics, use git to version control their software, and GitHub to make it available to a wide audience. This is important so other people can see what you are working on, and can lead to collaborations in longer term. The link at the top right of this page will take you to the home of this website, which is version controlled using git and published to the world using GitHub.

Installing git and joining GitHub

Depending on which machine you have (Windows, Mac, Linux), you can follow the instructions on the official git website to install it. Once installed, open up terminal on a Mac or Linux, or powershell on Windows, and type:

git --version

On Windows machines, I'd recommend installing WSL (Windows Subsystem for Linux), which should have git installed on it already. Failing that, git BASH is a good alternative. Install that here. Provided that you are not crippled by the security system of your computer, this should be relatively easy to do.

Once you have installed git, we can look at GitHub. You can sign up for a free account on the GitHub website. It should be very self-explanatory to sign up.

Creating a new repository

There are two ways to create a new repository. You can create a new repository on GitHub and clone it to your local machine, or you can create a new repository on your local machine and push it to GitHub. I recommend the latter, as I think it's easier to do.

You firstly need to create a directory/folder on your machine. In terminal or powershell, navigate to the directory and type: 

git init

This will create a new git repository in your current directory. It does this by creating a hidden folder. Now you can add, modify, or delete files in the directory and they can be tracked.

Now go over to GitHub and create a new repository. It will ask for some details, so fill those in. Once you click 'create repository', there will be some commands you can run to link your repository you made on your computer just now, to the remote one.

The code will be something like the following:

# push an existing repository from the command line
git remote add origin https://github.com/USERNAME/REPO NAME.git
git branch -M main
git push -u origin main
